## 🌐 2026-03-27 下午 - 瀏覽器工具使用策略

**決策：** 優先使用 OpenClaw 內建 `browser` 工具，減少使用 API/爬蟲方式。

**原因：**
- `browser` 工具更安全（內建認證）
- 可以處理需要登入的網站
- 可以處理 JavaScript 渲染的頁面
- 可以截圖、點擊、輸入等互動操作
- 不會被反爬蟲機制封鎖

**使用優先級：**

| 優先級 | 工具 | 使用場景 |
|--------|------|----------|
| 🔝 **首選** | `browser` | 複雜頁面、需要登入、互動操作 |
| 🟡 **備用** | `web_fetch` | 靜態文字頁面、簡單文章 |
| 🔻 **少用** | `web_search` | 快速搜尋摘要 |
| ❌ **避免** | 自寫爬蟲 | 不穩定、易被封鎖 |

**標準流程：**
1. 先用 `browser` 開啟網頁
2. 截圖確認頁面正確
3. 用 `browser` 提取內容
4. 必要時點擊/導航到下一頁
5. 整理資料給用戶
